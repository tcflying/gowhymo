# gowhymo
第一次运行只要输入昵称就能生成一个本地账户。
如果需要云同步，就需要通过手机注册一个账户，然后在设置中开启云同步。

在身份系统中，地址、公钥和私钥各有不同的用途：

地址（Address）
作为主键ID：用于标识用户身份，存储在数据库中
公开可见：可以安全地分享给他人
数据关联：所有用户数据都通过这个地址进行关联
公钥（Public Key）
验证签名：用于验证用对应私钥创建的签名
加密数据：他人可以用你的公钥加密数据，只有你的私钥能解密
身份验证：证明某个地址确实属于你（通过密码学证明）
私钥（Private Key）
创建签名：对数据进行数字签名，证明数据确实来自你
解密数据：解密用你公钥加密的数据
身份控制：拥有私钥就等于拥有对应地址的控制权
实际应用场景
数据签名：用私钥签名本地数据 → 用公钥验证签名真实性
多设备同步：在新设备上输入私钥 → 恢复相同身份和地址
数据加密：敏感数据可以用公钥加密 → 只有私钥持有者能解密
身份证明：通过签名挑战来证明"我是地址的主人"
这样设计既保证了数据的安全性（私钥不传输），又实现了身份的可验证性（公钥公开）。
